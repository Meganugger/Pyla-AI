Welcome to the Pyla AI Documentation

What is Pyla AI?

Pyla AI is a bot that plays Brawl Stars for you without you having to do anything. It can help you farm mastery and gain trophies up to a certain point.

See Pyla AI in Action

Video of Pyla AI in action

  • Computer running Windows 10/11
  • Python 3.11.9
  • LDPlayer/Bluestacks (LDPlayer recommended)

Available Gamemodes

  • 3v3 gamemodes
  • 5v5 gamemodes

Getting Started

In this section, we will guide you through the initial setup process and how to use Pyla AI.

Downloading Resources

Before you can use Pyla AI, you need to download and install the following resources:

Install Requirements

  • Computer running Windows 10/11
  • Python 3.11.9
  • LDPlayer/Bluestacks (LDPlayer recommended)

1. Python 3.11.9

To get started, download Python 3.11.9 (required for running the Pyla AI program) from the Official Python Website.

2. Android Emulator

Next, download LDPlayer or Bluestacks (preferably LDPlayer), which is the Android emulator responsible for running Brawl Stars: https://www.ldplayer.net/

3. Pyla AI

Finally, download Pyla AI using the Download button in the navigation bar.

Setting up Pyla AI

Once you have downloaded those files, you would need to install Python 3.11.9 and LDPlayer/Bluestacks, as well as configuring the Pyla AI files.

Install Python 3.11.9

After downloading the Python 3.11.9 installer, double-click to open it. You will see the Python 3.11.9 setup window. Make sure you select "Use Admin Privileges" and "Add python.exe to path". You can now select "Install Now". After installation, close the window.

LDPlayer

After you have downloaded LDPlayer 9, double click to open it. You will see an LDPlayer window with an install button. Click the install button, and wait for it to finish. After installation, close the window. LDPlayer should launch for you.

Pyla AI

Once Pyla AI is downloaded, unzip the file by double-clicking on it. You will see many different folders in the .zip file. Make a new folder, and name it something you will recognize as Pyla AI, for example, "Pyla AI". Copy everything from the .zip you downloaded, using Control + A to move it to the folder you have just created.

Starting Pyla AI

Starting Pyla for the first time

  • It will download the models required for running Pyla AI
  • It may fetch updates from the Brawl Stars API for new brawlers

Running Pyla AI

Before doing anything, you must set your display screen zoom to 100% in your PC settings. You can change it back when you are not using Pyla AI. After changing zoom settings, double click the "start.bat" file in the folder you have made for the Pyla AI files. It will open cmd and install the models, as well as running the app.

Note: There will be some error messages upon the beginning of running the program. They aren't important and can't be hidden, so please ignore them.

How to use Pyla AI

Pyla Hub app

The start.bat will open a window called "Pyla Hub". This is the main control center where you can:

  • Configure the bot with your preferred gamemode
  • Select which emulator you're using
  • Access additional customization settings
  • Set timers for automated operation
  • View match history and win rates for brawlers the bot has played

LDPlayer

Pyla AI needs you to run Brawl Stars on your PC, which is where LDPlayer comes in. LDPlayer is an Android Emulator that creates a virtual Android device on your computer. You'll need to install Brawl Stars from the Play Store within this emulator to allow the bot to interact with the game.

Instructions

  1. Initial Setup: In the "Overview" tab of Pyla Hub, select your map orientation, gamemode, and emulator. You can also change the mastery madness amount according to the current event boost in the Additional Settings tab. Once everything is configured, click the "Start" button.
  2. Brawler Selection: Pyla Hub will launch another window called "PylaAI". This is where you'll select which brawler you want the bot to play. You can add multiple brawlers to form a queue - the bot will automatically switch to the next brawler once the current one reaches its target goal.
  3. Enter Brawler Data: A new window called "Enter Brawler Data" will appear. Here you'll need to input:
    • What you want the bot to push (Trophies or Mastery)
    • The target amount of trophies/mastery to reach
    • The brawler's current trophies
    • The brawler's current mastery level
    • The brawler's current win streak
  4. Access Brawl Stars: While the "Enter Brawler Data" window is open, switch to LDPlayer. If you haven't already, open the Play Store, sign in with your Google Account, and download Brawl Stars.
  5. Check Brawler Stats: Open Brawl Stars and sign in with your Supercell account. Navigate to the brawler you want to push and check its current trophies, mastery, and win streak.
  6. Complete Setup: Return to the "Enter Brawler Data" window and input the values you just checked. You can uncheck "Auto-pick Brawler in game" if you prefer to manually select your brawler instead of having the bot navigate to it automatically.
  7. Configure Emulator Keys: Before starting the bot, you need to set up the keymapping in your emulator:
    • Q on the Lobby's "Start" Button
    • Note: It must also be on the exit match button otherwise the bot won't leave matches
    • E (e) on the Super button (NOT F (f))
    • H on the Hypercharge Button
    • G on the Gadget Button
    • The movepad on the Movement joystick
  8. Start the Bot: Click "Submit" to close the data entry window. In the "PylaAI" window, click the "Start" button to begin the bot operation.
Note: If you don't see the "Start" button, check if you've resized the window to be smaller than its original size. If the window appears cut off, you may need to adjust your PC's display zoom settings to 100%.

Troubleshooting

Solutions to common issues and problems.

Common Issues

Bot won't move or attack:

Make sure you have configured the bot correctly.

  • Screen zoom: Make sure you have set the screen zoom to 100%, in your PC display settings.
  • Emulator keys: Make sure you have set up the keymapping (emulator keys) for the bot:
    • Q on the Lobby's "Start" Button
    • Note: It must also be on the exit match button otherwise the bot won't leave matches
    • E (e) on the Super button (NOT F (f))
    • H on the Hypercharge Button
    • G on the Gadget Button
    • The movepad on the Movement joystick
Note: Be sure to have the buttons at their original position when using Pyla (In training camp - Edit Controls → Reset)

Bot Info

This section provides detailed technical information about how Pyla AI works, by iyordanov.

Models

Pyla AI uses several specialized AI models to perform different functions within the game. Below is an explanation of each model and how it was trained.

Note: The model documentation is based on the brawl-stars-bot repository.

Specific in-game data

  • Detects: 79 brawlers + 750 skins + each skin/brawler abilities + 5 types of balls + gems + power cubes + poison clouds + walls + bushes
  • Reliability: 5.5/10
  • Maintenance: Very high

The assets were 107,395 images and the dataset was 100k images.

Why so much?

Each brawler has around 40 angles from which you can see them, plus extra angles when shooting, and extra angles while moving and shooting. This quickly stacks up.

How did I manage to get all the assets:

There are no assets for this online so I had to the following:

  1. Code a script which goes into a game and makes screenshots of each angle of the brawler.
  2. Train a model to crop out the brawler from the image
  3. Train a model to segment the brawler out of the image
  4. Get other in-game assets - Will skip this as it's just me manually going over every single map and becoming a Photoshop dev
  5. Build a dataset generator with the assets

Goal

Understanding the purpose and motivation behind Pyla AI.

Purpose

The goal of Pyla AI is to help players progress faster before they quit the game due to low-level brawlers making them lose every game.

The Progression Problem

For those unfamiliar with Brawl Stars, here's the issue:

Each brawler has two primary stats: Health and Attack. The leveling system works as follows:

  • Level 1 → 9: 3,090 coins
  • Level 9 → 11: 4,675 coins
  • Level 7 unlocks gadgets: 1,000 × 2 coins
  • Level 9 unlocks star powers and gears: 2,000 × 2 coins, 1,000 × 6 coins
  • Level 11 unlocks hyper charges: 5,000 coins

To have a maxed account (all brawlers at level 11 with at least 1 star power, 1 gadget, and 2 gears), you need:

1,046,730 coins

Free-to-Play Progression

An active free-to-play player will have the following monthly rewards:

3 Star Drops a Day

  • Progression: Average 7,740 coins
  • Time to complete: Average 12 hours

Complete Brawl Pass

  • Progression: 8,000 coins
  • Time to complete: 30 hours

Farm Mastery

(Possible for 2 months and 6 days)

  • Progression: 86,250 coins
  • Time to complete: 120 hours

Conclusion

If a player spends roughly 35 hours a month planning perfectly what and how to play, and is at 56,700 trophies, they will gain:

(7740+8000)×1.2 + 1863 + [(86250/120) × 35] coins a month, which is:

20,751 + 25,156 (the first 2 months).

Time to Max Account

This means it would take 4 YEARS for a free-to-play player to max their account.

Motivation

Pyla AI was created because the amount of grind is absurd. It takes 35 boring hours each month to grind, and then you can actually have fun. Many players don't have that kind of time. The only solution for busy people is to buy the Brawl Pass Plus, play 7 hours a month, and spend 3-4 hours on the last day completing quests.

Code

Understanding the technical implementation of Pyla AI.

This documentation aims to give you a rough idea of how the bot works without showing too much code, as it is private, but still giving you enough information to understand its functionality.

Navigation

To understand the code in the best way, explore these components in order:

  • Graph - Diagram showing the code logic flow
  • State finder - Identifies the current game state
  • State manager - Automates actions based on the current game state
  • Progress Observer - Keeps track of progress after each game to swap brawlers once specific goals are reached
  • Lobby automator - Automates more complicated actions inside the lobby
  • Detect - Transforms YOLOv8 output into an easier to use format
  • Movement - Finds the best movement with the detected data
  • Play - Main playing logic that inherits Movement and adds extra functionality
  • Time Management - Helps determine whether time thresholds have been reached for various activities
  • Main - Core component that initializes all classes and controls how everything works together
  • Screenshot Taker - Captures screenshots for the bot to process

State finder

Purpose: Identify the current game state

Used in: State Manager

How it works:
It uses template matching to compare parts of the screen to predefined images to identify specific game states. This method is quick and efficient because the lobby layout is static and rarely changes.

This is how the lobby looks after a game is over, and it hasn't changed in years. So we can find the state is 'end_game_lobby' by looking for the red thumbs down button.

Time Management

Purpose: This class helps in determining whether specific time thresholds have been reached for various activities or states, ensuring that actions are taken at appropriate intervals.

Used in: Main

How it works:
It keeps a record of the last time each action happened and checks if enough time has passed to perform the action again. If the time threshold is met, it updates the record to the current time.

Main

Purpose: The Main class is the core part of the application. It initializes all classes, loads the models, and controls how everything works together.

The bot plays brawlers in the order the user picked them. Once the wanted progression is reached, it will move to the next brawler.

Screenshot Taker

Purpose: Return an image of the screen in PIL format

Used in: LobbyAutomation, Main

How it works:
Initializes with a "camera" object from the "DXCam" library. Sometimes "DXCam" returns None images, which breaks the code. The take method ensures the image cannot be None.
